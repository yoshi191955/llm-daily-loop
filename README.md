# llm-daily-loop — LLM日足ループ（クラウド版・PCオフでも稼働）

カタリスト駆動のLLM予測を、GitHub Actions で**毎営業日**自動実行する。
**1日/3日の2つのhorizonを併走**し、どちらが優れるかを同一サンプルで比較する。

> ペーパー（名目）検証のみ。発注は一切行わない。投資助言ではない。

## なぜこの設計か（実証済みの根拠）

| 検証 | 結果 |
|---|---|
| テクニカル（SMA/ブレイクアウト・日足10年） | edge なし |
| 分散マルチアセット・トレンド | edge なし |
| 機械的モメンタム（15分足・2h・12,492件） | **的中50.4%＝コイン投げ、t=−58.84** |
| **LLMカタリスト予測（日足・42件）** | **ベータ分離合格・付加価値+1.05%/トレード・down予測8/10** |

→ **テクニカルはシグナルに使わない。LLMはカタリスト解釈に集中。**
→ 往復0.5%コストを超える値幅が要るため、**日足が下限**。3日足も併走して比較する。

## セットアップ

1. GitHubに Public リポジトリを作成し、このフォルダの中身をpush
2. **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `ANTHROPIC_API_KEY`
   - Secret: あなたのAnthropic APIキー（[console.anthropic.com](https://console.anthropic.com) で発行）
   - ※キーはコードにも README にも書かない。Secretsにのみ登録する
3. **Settings → Actions → General → Workflow permissions** → **Read and write**
4. **Settings → Pages** → Branch `main` / フォルダ `/docs`
5. Actions → `llm-daily-loop` → **Run workflow** で手動テスト

以降、**平日 12:30 UTC（米国の寄り付き前）に自動実行**。

## API費用（実測ベースの試算）

1回=52銘柄一括（入力 約22,300 / 出力 約4,160 tokens）

| モデル | 1回 | 1ヶ月(21営業日) |
|---|---|---|
| Haiku 4.5 | $0.043 | **$0.91** |
| **Sonnet 4.6（既定）** | $0.129 | **$2.72** |

安く抑えたいなら workflow の `CLAUDE_MODEL` を `claude-haiku-4-5-20251001` に変更。
GitHub Actions / Pages / yfinance は**無料**。

## ログの形（1予測 → 2行）

正本は「1行＝1(予測×horizon)」の**長形式マスターログ1枚**（シートは分けない）。

| 列 | 意味 |
|---|---|
| `horizon` | `1d` / `3d` |
| `lean` | **全52銘柄**に必ず付ける方向の傾き → クロスセクション検定用 |
| `score` | 確信度 0-1 |
| `direction` | **実際に賭けるか**（up/down/**flat**）。`score < 0.60` は flat＝見送り |
| `corr_bucket` | 相関バケット |

**`direction` と `lean` は役割が違う。** direction は選別（見送りこそ価値）、lean は市場中立の検定用。

## 安全設計

- **後知恵バイアス防止**: 寄り付き前にのみ予測を作る。当日の足が既に存在するなら予測しない（三重ガード：平日 / 寄り前 / 足未生成）
- **捏造しない**: データが取れない・満期前の行は `open` のまま残す
- **コストモデル**: 往復0.5% + 利益のみ25%課税（緩めない）
- **秘密情報**: `ANTHROPIC_API_KEY` のみ、環境変数から読む。コード・ログに残さない

## 分析

```bash
python analyze_predlog.py docs/predictions_log.csv
```

4階層で評価（下ほど厳しい）:
1. トレード単位（過大評価されがち）
2. **セッション単位**（同日の相関を補正した"本当の"標本）
3. **ベータ分離**（AlwaysUp比。「上げ相場に乗っただけ」を除外）
4. **クロスセクション**（同日の lean=up群 − down群。市場ベータが設計上ゼロ）

## 判定の一線（守ること）

**独立30セッション／100トレード**に達し、**セッション単位のt値がプラスで有意**、
かつ**ベータ分離とクロスセクションの両方で付加価値**が確認できるまで、**実弾には進まない**。
現状は 7セッション・セッション単位 t=−0.08（有意性ゼロ）。
毎営業日回せば約1.1ヶ月で判定可能。
