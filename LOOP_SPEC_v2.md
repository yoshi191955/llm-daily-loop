# LLM日足ループ v2 仕様書（1日/3日 併走・全52銘柄）

## 変更の要点
1. **対象を33社 → 52社**（`watchlist.csv`・相関バケット付き）
2. **全52銘柄を毎セッション予測**（予測数に上限は設けない。上限は将来の"発注"にのみ適用）
3. **1日/3日 の2つのhorizonを併走記録**（予測は1回。採点だけ2回＝追加コストほぼゼロ）
4. **全銘柄に `lean` と `score` を付与**（見送り銘柄にも）→ クロスセクション検定を可能にする

---

## 1. 行の構造（1予測 → 2行）

**正本は「1行＝1(予測 × horizon)」の長形式マスターログ1枚**。シートは分けない。
同じ予測から **horizon=1d の行** と **horizon=3d の行** を2行出力する。

```
pred_id            : C-0293-1d / C-0293-3d   （末尾に horizon を付けて一意化）
horizon            : "1d" または "3d"
```

これで既存の分析は `horizon` 列で絞るだけで動く。1日と3日を**同一サンプル上で公平に比較**できる。

## 2. 追加する列（3列）

| 列 | 意味 | 誰に付けるか |
|---|---|---|
| `lean` | 方向の傾き（`up` / `down`）。**必ずどちらかを付ける** | **全52銘柄**（flat含む） |
| `score` | 確信度 0.00〜1.00 | **全52銘柄**（flat含む） |
| `corr_bucket` | 相関バケット（watchlist.csv から転記） | 全52銘柄 |

**なぜ flat にも lean が要るか**：
同じ日の「upと傾いた群」と「downと傾いた群」のリターン差を見れば、
**市場全体の動き（ベータ）が構造的に相殺**される（＝市場中立の検定）。
「上げ相場に乗っただけ」という疑いを、統計処理ではなく**設計で**排除できる。
賭けない銘柄も、この検定には貴重なデータになる。

## 3. `direction` と `lean` の役割分担（重要）

- **`direction`**（`up`/`down`/**`flat`**）… **実際に賭けるか**。閾値未満は `flat`＝見送り。
  → **選別性を維持する**（292回中250回見送る、というこのシステムの core は変えない）
- **`lean`**（`up`/`down`）… **全銘柄の方向の傾き**。賭けるかどうかとは無関係。
  → クロスセクション検定にのみ使う

**この2つを混同しないこと。** direction を全銘柄に強制すると選別の価値が消える。

## 4. 採点ルール

| horizon | エントリー | 決済 | 備考 |
|---|---|---|---|
| **1d** | session_open | **当日の引け** | 現行と同じ |
| **3d** | session_open | **3営業日後の引け** | 祝日・休場をまたぐ場合は営業日で数える |

コストモデルは両方とも現行のまま：**往復0.5% ＋ 利益のみ25%課税**。
```
net_return_pct = (return_pct - 0.5) * 0.75   if return_pct > 0.5
               = (return_pct - 0.5)          otherwise
```

3d行は満期が来るまで `status=open` のまま。**必ず後日きちんと採点すること**
（未採点のまま放置するとサンプルが失われる）。

## 5. 予測数に上限は設けない

- 毎セッション、**52銘柄すべてに行を作る**（×2 horizon = 104行/セッション）
- `direction` が up/down になる数（＝実際の賭け）は、確信度しだいで自然に決まる。**上限を課さない**
- **理由**：今はペーパーでリスクがゼロ。データは多いほど良い。
  同日内トレードの相関は「データを減らす」のではなく「**セッション単位のt値で補正**」して対処する
- **上限が必要になるのは実弾を張る時だけ**。その時は「予測52 → 発注は上位5」とする
  （相関バケットにつき最大1社、同日合計エクスポージャー上限も併用）

## 6. 運用で最優先に直すこと

**毎営業日、確実に走らせる。** 実績は25日間で7セッションのみ（6/24〜7/6 に11日の空白）。
- 現ペース(約2回/週) → 独立30セッションまで **約2.7ヶ月**
- 毎営業日(5回/週)   → **約1.1ヶ月**

**判定が2.5倍速くなる。今いちばん効くレバーはこれ。**
なぜ11日空いたのか（スケジュール停止／実行エラー／データ取得失敗）を必ず特定すること。

## 7. 変えてはいけないもの

- **後知恵バイアス防止ガード**（寄り付き前にのみ新規予測を作る）
- **選別**（`flat`＝見送りを恐れない。賭けない判断こそ価値）
- **コストモデル**（0.5%＋税25%。緩めない）
- **捏造しない**（取得できない値は空欄のまま）
- **LLMをテクニカル指標の計算に使わない**（機械の仕事。LLMはカタリスト解釈に集中）
